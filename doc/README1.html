Part 1 Overview Simple Case

JUnit is a unit testing framework to write repeatable tests. JUnit contains many annotations that are used while creating test cases.

Some of the most basic annotations are as below.

  • @BeforeClass: It is used to write code that we want to run before all test cases.
  • @Before: It will run before every test case.
  • @Test: This is actual test case.
  • @After: It will run after every test case.
  • @AfterClass: It is used to write code that we want to run after all test cases.

 

The first example shows an example of a basic set of tests cases using basic annotations. The case shows a two charging implementations, extending a charging interface. Separate test cases exist for each case, with annotations for the lifecycle cases above. A test suite exists as shown below. The outputs of the 2 classes implementing the interfaces is then tested.

Part 2 More Complex Case and TDD Process

The second example shows a case of Test Driven Development, where the test cases are defined prior to the code. In this case a typical development cycle may be

  • an interface is defined first
  • then a test case to exercise the interface
  • a rudimentary failed class (without implementation)
  • a test case is then run

 

The model chosen here is a customer, who has a set of transactions associated with them. There is a customerService interface and a customerServiceImplementation of that interface.

 

The test methods also show the use of Java Streams to filter through a set of transactions and show the right values are returned.

 

Junit has a set of assertions based on the type of comparison - for example compare two strings, or show that they are not equal.

 

See package testDrivenDevelopment
